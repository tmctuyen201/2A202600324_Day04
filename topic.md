Kiến Trúc Hệ Thống TravelBuddy AI AgentTài liệu này mô tả cách xây dựng một trợ lý du lịch thông minh sử dụng LangGraph cho logic xử lý (Backend) và Streamlit cho giao diện người dùng (Frontend).1. Tổng Quan Kiến TrúcHệ thống được thiết kế theo mô hình Agentic Workflow, nơi AI không chỉ trả lời câu hỏi mà còn có khả năng tự quyết định sử dụng các công cụ (tools) để giải quyết yêu cầu phức tạp của người dùng.Các thành phần chính:Frontend (Streamlit): Cung cấp giao diện chat, tiếp nhận Input từ người dùng và hiển thị luồng suy nghĩ (logs) cùng câu trả lời cuối cùng của Agent.Backend (LangGraph): Điều phối luồng điều khiển (Control Flow) giữa LLM và các Tools.LLM (GPT-4o-mini): "Não bộ" đưa ra quyết định dựa trên system_prompt.txt.Mock Database & Tools: Giả lập dữ liệu chuyến bay, khách sạn và tính toán ngân sách.2. Cấu Trúc Thư Mục Dự ÁnPlaintextlab4_agent/
├── .env                # Chứa OPENAI_API_KEY [cite: 17]
├── system_prompt.txt   # Định nghĩa Persona và Rules của Agent [cite: 28]
├── tools.py            # Định nghĩa các hàm: search_flights, search_hotels, calculate_budget [cite: 61]
├── agent_graph.py      # Xây dựng StateGraph của LangGraph [cite: 283]
└── app.py              # Frontend Streamlit (Giao diện chính)
3. Thành Phần Backend (LangGraph)Backend sử dụng một StateGraph để quản lý trạng thái hội thoại.Các Node chính:agent_node: Tiếp nhận tin nhắn, gắn System Prompt và gọi LLM để quyết định xem có cần dùng tool hay không.tools_node: Thực thi các hàm Python thực tế nếu LLM yêu cầu gọi tool.Luồng logic (Edges):START $\rightarrow$ Agent.Agent $\rightarrow$ Tools (nếu LLM yêu cầu gọi hàm).Tools $\rightarrow$ Agent (để Agent tổng hợp kết quả sau khi có dữ liệu từ tool).Agent $\rightarrow$ END (khi Agent đã có đủ thông tin trả lời người dùng).4. Thành Phần Frontend (Streamlit)Streamlit đóng vai trò là lớp giao diện chạy trên trình duyệt:Chat Interface: Sử dụng st.chat_message và st.chat_input để tạo trải nghiệm giống ChatGPT.Session State: Lưu trữ lịch sử tin nhắn (messages) để duy trì ngữ cảnh cuộc hội thoại.Agent Execution: Khi người dùng gửi tin nhắn, Streamlit sẽ gọi hàm graph.invoke() từ Backend và hiển thị kết quả.5. Các Công Cụ Giả Lập (Tools) Hệ thống tích hợp 3 công cụ chính để Agent có thể "làm việc":search_flights: Tra cứu chuyến bay từ FLIGHTS_DB (Hà Nội, Đà Nẵng, Phú Quốc, HCM).Google Hotels: Tìm khách sạn theo thành phố và lọc theo ngân sách tối đa.calculate_budget: Parse chuỗi chi phí và tính toán số dư thực tế, đưa ra cảnh báo nếu vượt ngân sách.6. Hướng Dẫn Chạy LocalCài đặt thư viện:Bashpip install langchain langchain-openai langgraph streamlit python-dotenv
Cấu hình API Key: Tạo file .env và dán OPENAI_API_KEY của bạn vào.Khởi chạy ứng dụng:Bashstreamlit run app.py
